# 常见问题

## 模板声明

**Q: 如何声明只读结构体的模板？**

A: 只读结构体需要匹配的构造器。SG 自动搜索与字段名和类型匹配的构造器，通过构造器创建实例。

**Q: 模板中能用自定义类型吗？**

A: 能。给自定义类型加 `[Template]` 属性，嵌套使用即可。SG 自动解析依赖顺序。

**Q: 如何在 BCL 类型上使用模板？**

A: 用 `[ExternalTemplate(typeof(List<>), "...")]` 覆盖默认模板。类级覆盖优先级高于接口默认模板。

## 运行时

**Q: TryGet 返回 false 怎么办？**

A: 检查类型是否标注了 `[Template]`。检查是否有 `[TemplateIgnore]` 标记了所有字段导致空模板。

**Q: 扫描失败返回 0？**

A: 输入与模板格式不匹配。检查分隔符、字段类型、可选块位置。用诊断工具对比模板和输入。

**Q: 必须缓存 ISerializerBlock 吗？**

A: 非必须。`TryGet<T>` 是静态字段读取，开销极低（~2ns）。但缓存到 `static readonly` 字段更友好。

**Q: 如何序列化 List 等集合？**

A: `List<T>` 等集合有默认接口模板，直接使用 `ISerializerBlock<List<T>>` 即可。格式为首元素无分隔符、后续逗号空格分隔。

## 泛型

**Q: 自定义泛型类型的模板怎么写？**

A: 用类型参数名作为占位符：`[Template("<T Value>")]`。使用方引用具体实例如 `Wrapper<float>` 时自动合成。

**Q: 为什么我的自定义集合类型不工作？**

A: 建议为集合定义接口并标注模板，而不逐个标注具体类。所有实现该接口的类型自动获得模板。

**Q: 如何在文本中使用枚举名而非整数？**

A: 在枚举成员上加 `[Tag("fire")]` 标签，模板中直接使用枚举类型名（如 `<Element Elem>`）。SG 自动生成标签到枚举值的双向映射。无 Tag 的枚举值在 Emit 时回退到 `value.ToString()`。

**Q: 如何为字段类型起别名？**

A: 使用 `[assembly: TypeAlias("HP", "float")]` 程序集级声明。模板中写 `<HP Health>` 即可，解析行为与原类型一致。别名可映射到任何已注册类型（内置和自定义均可）。
